##### **1.Abstract**



This project demonstrates a methodology for detecting Domain Generation Algorithms (DGA) and suspicious DNS traffic. By utilizing Python-based statistical modelling (Shannon Entropy) and lexical analysis, we successfully built a tool that identifies potential Command and Control (C2) domains without relying on traditional, easily bypassed blacklists.



##### **2.Problem Statement**



Modern malware uses DGAs to generate thousands of random domains to communicate with attackers. Static firewalls fail to block these because the domain names change constantly.



**•The Goal:** Build a system that "looks" at the structure of a domain name to determine if it was created by a human (Benign) or an algorithm (Malicious).



##### **3.Methodology \& Technical Implementation**



###### **3.1 Tools Used**



**•Linux (Ubuntu):** Host environment for secure traffic analysis.

**•Python 3:** Used for the primary detection engine.

**•Pandas:** For data organization and reporting.

**•TLDExtract:** To isolate the "core" domain from subdomains.
**Evidence:** fg1


###### **3.2 Detection Logic: Shannon Entropy**



The core of our detection is the Shannon Entropy formula. It measures the "randomness" or "uncertainty" of characters in a string.

**•Benign domains** (*e.g., google*) have lower entropy because they follow natural language patterns.

**•DGA domains** (*e.g., xy12234lkjndf*) have high entropy because the character distribution is highly randomized.





##### **4. Experimental Results**



The system was tested against a mixture of known safe traffic and simulated malicious strings. The following table represents the actual output generated by the implemented system:



|**DOMAIN**|**ENTROPY SCORE**|**CLASSIFICATION**|
|-|-|-|
|google.com|1.92|Benign|
|Wikipedia.org|2.64|Benign|
|bankofamerica.com|3.33|MALICIOUS (DGA)|
|xy12234lkjndf.net|3.55|MALICIOUS (DGA)|
|vbn9988zz.top|2.50|MALICIOUS (DGA)|





###### **4.1 Result Analysis:**



The system successfully isolated the high-entropy malicious string. However, it flagged bankofamerica.com as suspicious due to its length. This highlights the need for a Whitelist in professional deployments to reduce "False Positives."

**Evidence:** fg2, fg3, fg4

##### **5. Deployment and Countermeasures**



Once the script identifies a malicious domain, the following defensive steps are recommended:



**1.DNS Sinkholing:** Redirecting malicious domains to a safe internal IP.

**2.SIEM Alerting:** Sending the flagged logs to a security dashboard for an analyst to review.

**3.Automatic Blocking:** Updating firewall rules via API to block the specific IP generating high-entropy queries.



##### **6. Conclusion**



The project confirms that lexical analysis is a viable method for identifying zero-day threats. By measuring entropy and digit density, we can detect malware communication even when the attacker uses previously unknown domain names. Future improvements would include integrating a Random Forest machine learning model to improve accuracy.
